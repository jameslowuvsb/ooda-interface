## WorldView — Technical Architecture

### Core Stack
| Component | Technology |
|-----------|-----------|
| **3D Globe Engine** | CesiumJS + Google Photorealistic 3D Tiles API |
| **Frontend Framework** | Next.js 16 + React |
| **Styling** | Tailwind CSS v4 |
| **Visual Modes** | Night vision, FLIR thermal, CRT scan lines, spy telescope viewport |
| **AI Models** | Google + Anthropic models (multi-agent orchestration) |

### Data Layers (45+ confirmed)
| Category | Source | Data Type |
|----------|--------|-----------|
| **Aviation** | OpenSky Network | Live commercial flight positions |
| **Aviation** | ADS-B Exchange | Crowdsourced military flight tracking |
| **Maritime** | AIS beacons | Vessel position, heading, speed, flag state |
| **Satellites** | CelesTrak TLE | Real-time orbital positions of all tracked satellites |
| **Seismic** | USGS feeds | Live earthquake data |
| **Surveillance** | Public CCTV feeds | Camera feeds draped onto 3D city models |
| **Weather** | Weather radar | Precipitation overlays |
| **Space** | Asteroid trajectories | Near-earth object tracking |
| **SIGINT** | GPS jamming detection | GPS interference zones |
| **Geopolitical** | Airspace closures | No-fly zones across 9+ countries |

### How It Was Built
- **Timeline**: 3 days (but built on 6 years of Google spatial computing expertise)
- **Method**: AI agent swarm — up to 8 agents running simultaneously
  - Agent 1: Satellite tracker subsystem
  - Agent 2: CCTV feed wiring
  - Agent 3: Shader pipeline (night vision, FLIR, CRT)
  - Agent 4-8: Other subsystems (aviation, maritime, UI, etc.)
- **Workflow**: Described features in voice notes + screenshots → fed to agents → steered results
- **Key Insight**: "Power is defined not by who owns data but by who can combine it fastest"

### Dual-Engine Mapping (from community OSS version)
- **3D Globe**: globe.gl for photorealistic rendering
- **2D Flat Map**: deck.gl (WebGL) for flat map analysis
- 45 data layers persist across both views

### OSS Architecture Details (jedijamez567/worldview_oss)
- **State Management**: Zustand
- **Graph Database**: JanusGraph client
- **UI Layout**: 3-column layout with scope/telescope CSS
- **API Routes**: Next.js API routes for data proxying
- **Agent System**: Mission Control hooks + agent swarm operations
- **News Sources**: GDELT, RSS feeds (BBC, Al Jazeera), Reddit
- **Camera Feeds**: 10+ Department of Transportation camera APIs
